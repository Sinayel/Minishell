minishell notes

shell :
	les commandes sont dans /bin
	interactive mode :
		accepte les inputs tapes depuis le clavier
	non interactive :
		le shell execute une commande depuis un fichier qu'il lit
	
	le shell attend que les commandes synchronisees soient totalement executes avant d'accepter d'autres inputs

builtin : 
	Une commande qui est implementee en interne par le shell lui meme plutot qu'un executable dans le fichier systeme

control operator :
	A token that performs a control function. It is a newline or one of the following: ‘||’, ‘&&’, ‘&’, ‘;’, ‘;;’, ‘;&’, ‘;;&’, ‘|’, ‘|&’, ‘(’, or ‘)’.

exit status : 
	The value returned by a command to its caller. The value is restricted to eight bits, so the maximum value is 255.

special builtin :
	A shell builtin command that has been classified as special by the POSIX standard.

Syntaxe du Shell :
	quand le shell reconnait un '#', il l'ignore ainsi que le reste de la ligne

	le shell lit l'input et le divise en mots et en operateurs, en utilisant les regles de quotation pour assimiler les differents mots et caracteres
	Ensuite, le shell decoupe ces infos en commandes et autres operateurs, en enlevant la signification de certains mots et caracteres, puis 
	redirige l'input/output si necessaire, execute la commande specifiee et attends son 'exit status' en le rendant (l'exit status) discornible pour 	
	une future inspection ou un traitement plus approfondi :

	1) lit l'input depuis un fichier, depuis une chaine fournie en argument ou depuis le terminal de l'utilisateur
	2) separe l'input en mots et oprateurs en obeissant aux regles de quotations, ces tokens sont separes par des metacaracteres, les alias sont 	traites par cette etape
	3) parse ces "tokens" en commandes simples et composees
	4) effectue les differentes extensions du shell, en divisant les tokens developpes en listes de filenames et en commandes et arguments
	5) effectue les redirections eventuelles et supprime les opperateurs de redirection et leurs operandes de la liste des arguments
	6) execute la/les commande.s
	7) attend la fin de la commande et collecte son exit status

quotation :
	elle est utilisee pour retirer la signification de certains caracteres ou mots au shell, par exemple il permet de retirer le traitement special au caracteres speciaux
		backslash `\` -> bash escape caractere, il preserve la valeur litterale du prochain caractere a l'exception d'une newline

		single quotes `'` -> preserve la valeur litterale de tous les caracteres a l'interieur des quotes, une single quote ne peut pas etre a
			l'interieur d'autres (meme precede d'un blackslash)

		double quotes `"` -> preserve la valeur litterale de tous les caracteres a 'interieur des quotes a l'excepetion de ‘$’, ‘`’, ‘\’. Le '!' n'a pas de signification particuliere
            une double quote peut etre inseree a l'interieur d'autres seulement si elle est precedee d'un backslash

        ANSI-C quoting :
            les sequences de caracteres sous la forme $'string' sont traites comme une forme speciale de simple quote :

            - \a -> alert (bell)
            - \b -> backspace
            - \e = \E -> an escape charactere
            - \f -> form feed
            - \n -> newline
            - \r -> carriage return  (machine a ecrire katching)
            - \t -> tab
            - \v -> vertical tab
            - \\ -> backslash
            - \' -> single quote
            - \" -> double   //
            - \? -> question mark
            - \nnn -> affiche le caractere correspondant a la valeur de n
            - \xHH -> ?
            - uHHHH -> ?
            - \UHHHHHHHH -> ?
            - \cx -> ?

Comments :
    dans un shell non-interactif, un mot commancant par '#' fait que le mot et tous les caracterees restants sont ignores
    dans un shell interactif, les commentaires sont actives par defaut (interactive_comments)

Commands :
	le type decommande rencontre le plus souvent est juste une serie de motes separes par des blancs et termine par un control operator
	le premier mot correspond generalement a la commande a executer et le reste des mots des arguments a cette commande 
	l'exit status d'une commande simple est apporte par la fonction waitpid() ou 128+n si la commande est terminee par un signal n

Lists of commands : (a gere si bonus)
	An AND list has the form
		$>"command1 && command2"
	command2 is executed if, and only if, command1 returns an exit status of zero (success).

	An OR list has the form
		$>"command1 || command2"
	command2 is executed if, and only if, command1 returns a non-zero exit status.

	The return status of AND and OR lists is the exit status of the last command executed in the list

			(expression )
                Returns the value of expression.  This may be used
                to override the normal precedence of operators.
            ! expression
                True if expression is false.
            expression1 && expression2
                True if both expression1 and expression2 are true.
            expression1 || expression2
                True if either expression1 or expression2 is true.

Expressions combinées
	Les expressions peuvent être combinées en utilisant les opérateurs suivants, décrits par ordre de priorité décroissante :

	( expression )
		Renvoie la valeur de l'expression. Cela peut être utilisé pour surpasser la priorité normale des opérateurs.
	! expression
		Vrai si expression est fausse.
	expression1 && expression2
		Vrai si expression1 et expression2 sont toutes les deux vraies.
	expression1 || expression2
		Vrai si expression1 ou expression2 est vraie.
	Les opérateurs && et || n'évaluent pas expression2 si la valeur de expression1 suffit à déterminer la valeur renvoyée par la condition tout entière.
